# CAL — Causal Architecture of Life

**CAL — Causal Architecture of Life** is the canonical English name of **生命因果架构**.

CAL is a causal framework for describing how living systems generate, stabilize, repeat, interrupt, and update behavior under continuity constraints.

Observable behavior is treated as a system output. It is generated by the interaction of the system's current state, historically formed structures, current information, continuity-directed constraints, and reality feedback.

## Canonical Public Definition

CAL studies the causal architecture that connects:

Current State  
→ Available Structures  
→ Continuity-Directed Constraint  
→ Causal Selection  
→ Observable Output  
→ Reality Feedback  
→ Structural Update

`Causal Selection` means the causal resolution produced by the system's active constraints. It does not mean an independent free-will operator outside the causal chain.

## Protected Boundary

This public repository does **not** disclose:

- equations, quantitative relationships, parameters, weights, or scoring rules;
- variable extraction, classification, matching, or inference procedures;
- template reconstruction, relationship matching, or prediction logic;
- prompts, agent orchestration, memory architecture, database schemas, APIs, or backend code;
- user data, private cases, product implementation, or operational workflows;
- the complete executable CAL model.

Absence from this repository does not mean a concept or mechanism is absent from CAL. It means the material belongs to the protected theory or Runtime layer rather than the public definition layer.
