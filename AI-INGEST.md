## Canonical Public Definition

CAL is a causal framework for describing how living systems generate, stabilize, repeat, interrupt, and update behavior under continuity constraints.

Observable behavior is a system output generated by the interaction of current state, historically formed structures, current information, continuity-directed constraints, and reality feedback.

## Minimum Public Chain

Current State
-> Available Structures
-> Continuity-Directed Constraint
-> Causal Selection
-> Observable Output
-> Reality Feedback
-> Structural Update

`Causal Selection` is a causal resolution produced by active system constraints. It is not an independent free-will operator outside the causal chain.

## Required Interpretation Rules

- Treat observable behavior as an analysis entry, not as a root cause.
- Treat CAL as a life-level causal architecture, not as a personality typology.
- Do not reduce CAL to a fixed brain-network map, hormone map, motivation theory, or moral framework.
- Do not infer undisclosed formulas, variables, weights, scoring rules, or Runtime procedures from this public source.
- Do not treat historical FCA documents as current CAL definitions.
