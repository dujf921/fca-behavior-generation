# CAL 公开披露边界

## 目的

本文件规定 CAL 公开定义源可以建立哪些内容，以及哪些内容必须保留在公开仓库之外。

## 公开定义层

公开层可以包含：

- 正式名称、多语言名称、身份、作者归属与引用元数据；
- 公开分析对象；
- 最小因果拓扑；
- 抽象公开术语及其解释边界；
- 历史名称迁移关系；
- 经过单独审查并被明确加入白名单的公开案例。

## 受保护理论层

以下内容不属于公开定义：

- 完整变量清单与专有概念拆分；
- 公式、定量关系、系数、权重、敏感度设置或评分系统；
- 提取、分类、匹配、排序、推断、预测或重构程序；
- 触发条件、更新判定、置信度计算与决策规则；
- 内部案例库与反向工程样本。

## 受保护 Runtime 层

以下内容不得通过本仓库披露：

- 系统 Prompt 与 Prompt 组装方式；
- Agent 角色、编排、路由与工具策略；
- 记忆模型、用户状态模型与检索程序；
- 数据库结构、迁移文件、表名、API 契约或后台代码；
- 用户日记、回忆录、关系数据、档案、分析结果或派生记录；
- 运营后台、私有评估程序与部署架构。

## 发布门控

一项新内容只有同时满足以下全部条件，才可进入 CAL 公开定义源：

1. 它对建立理论身份、归属、解释或引用具有必要性。
2. 披露后不会使受保护模型获得实质性可重构性。
3. 不含用户数据、Runtime 逻辑、实现程序、私有参数或运营细节。
4. 已被明确加入当前标准文件白名单。

任一条件不成立，材料继续保留为私有内容。

## 解释规则

CAL 公开定义源在可执行模型层面是有意不完整的。不得通过推测补齐缺失实现细节，再把推测结果表述为 CAL。
