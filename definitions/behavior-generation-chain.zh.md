# 行为生成链 | 人类行为 FCA

可观察输出  
-> 前台进入  
-> 历史模板匹配  
-> 新结构生成  
-> 执行维持  
-> 反馈更新

## 1）可观察输出

- Input: 行动轨迹、停滞、反复、中断、状态变化信号
- Process: 将可观察行为登记为分析入口
- Output: 可用于链路回溯的结构化观察集
- Failure Mode: 把输出当成最终原因，跳过链路定位

## 2）前台进入

- Input: 竞争进入处理前台的内外部信号
- Process: 基于显著性进行前台选择与优先级分配
- Output: 前台输入队列
- Failure Mode: 威胁或关系信号占满前台，压缩后续环节可用容量

## 3）历史模板匹配

- Input: 前台输入队列与历史模板
- Process: 用旧结构匹配并解释当前输入
- Output: 维持连续性的解释框架
- Failure Mode: 旧模板锁死，吸收新反馈并阻断更新

## 4）新结构生成

- Input: 解释框架、现实约束、未解决任务需求
- Process: 生成超出旧模板的新因果路径与新行动结构
- Output: 可执行的新路径候选
- Failure Mode: 长期解释与复盘，但无法产出新结构

## 5）执行维持

- Input: 已生成的新路径候选
- Process: 在重复负载下维持路径运行，形成持续现实输出
- Output: 稳定执行序列
- Failure Mode: 路径存在但在长期负载下塌陷

## 6）反馈更新

- Input: 现实执行结果与回流反馈
- Process: 更新下一轮的前台竞争、模板权重与路径选择
- Output: 已更新的下一轮生成条件
- Failure Mode: 反馈被旧模板吸收，系统回到旧循环
