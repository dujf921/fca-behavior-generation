## No One-to-One Variable Migration

The move from FCA to CAL is a change of model level, not a terminology substitution exercise.

Historical FCA systems must not be mechanically renamed as CAL variables. CAL's protected variables, formal relations, and Runtime procedures are not disclosed through this migration table.

## Canonical Rule

Use `CAL — Causal Architecture of Life` and `生命因果架构` for all current references.

Use historical FCA terminology only when describing repository history, prior publications, or migration provenance.
