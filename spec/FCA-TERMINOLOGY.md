# FCA Terminology Boundary for Behavior Generation

**Model:** Functional Causal Architecture for Behavior Generation (FCA)  
**Chinese:** 行为生成的功能因果架构  
**Version:** 2.0  
**Status:** Terminology boundary file  

---

## 1. Official Model Name

Official name:

```text
Functional Causal Architecture for Behavior Generation
```

Official abbreviation:

```text
FCA
```

Chinese name:

```text
行为生成的功能因果架构
```

Short Chinese reference:

```text
功能因果架构
```

The previous name **AFA Triplex Modulation Architecture (AFA-TMA)** is historical only.

Do not use AFA as the official model prefix in FCA 2.0.

Do not define FCA as a generic causal architecture. FCA 2.0 specifically refers to behavior generation.

---

## 2. Official Core Sequence

FCA 2.0 core sequence:

```text
SN → DMN → FP → ECN
```

Chinese:

```text
入口敏感度 → 叙事模板 → 方案生成 → 执行承重
```

This sequence is the minimum causal chain of behavior generation in FCA.

Any FCA explanation must preserve the four-system distinction.

## 4. FP Boundary

FP means:

```text
Forward Planning / Structured Simulation System
```

FP does not mean:

```text
Frontoparietal Network
```

FP is not claimed to be an independent anatomical brain network.

FP is a functional solution-generation layer.

It exists in FCA to separate:

```text
solution generation
```

from:

```text
execution control
```

Chinese:

FP 不是额顶网络的同义词，也不是独立解剖脑网络声明。

FP 是功能性的方案生成层，用于把“方案生成”与“执行控制”拆开。

---

## 5. ECN Boundary

ECN means:

```text
Executive Chain / Execution Stability System
```

In FCA, ECN controls execution sequence and execution persistence.

ECN does not generate structural solutions.

Invalid expression:

```text
ECN creates a new strategy.
```

Valid expression:

```text
ECN executes or fails to execute a strategy generated by FP.
```

Chinese:

ECN 不生成新方案。

ECN 负责把 FP 生成的方案转化为行动序列，并维持执行连续性。

---

## 6. DMN Boundary

DMN means:

```text
Narrative / Context Template System
```

DMN supplies templates.

DMN does not produce structured causal plans.

Invalid expression:

```text
DMN solves the problem.
```

Valid expression:

```text
DMN supplies narrative and contextual templates that FP may use during solution generation.
```

Chinese:

DMN 提供叙事、记忆、身份、关系和情境材料。

DMN 不生成结构化解决方案。

---

## 7. SN Boundary

SN means:

```text
Salience / Sensitivity Gate
```

SN controls foreground entry.

SN does not generate narratives, plans, or execution sequences.

Invalid expression:

```text
SN explains the event.
```

Valid expression:

```text
SN determines whether the event enters foreground processing.
```

Chinese:

SN 决定输入是否进入前台处理。

SN 不生成叙事、不生成方案、不执行动作。

---

## 8. Hormonal Modulation Terms

FCA may use these modulation terms:

- cortisol bias
- dopamine bias
- oxytocin bias

Chinese:

- 皮质醇偏置
- 多巴胺偏置
- 催产素偏置

These are modulation axes, not networks.

They must not be written as hard one-to-one mappings.

Invalid expression:

```text
Cortisol is DMN.
Dopamine is FP.
Oxytocin is ECN.
```

Valid expression:

```text
Cortisol bias may increase threat-related foregrounding and DMN threat-template weighting.
Dopamine bias may support FP simulation energy and solution search.
Oxytocin bias may indirectly support execution persistence through perceived safety and relational continuity.
```

---

## 9. Forbidden Primary Explanatory Variables

The following terms must not be used as primary causal variables in FCA:

- personality
- attitude
- emotion category
- feeling
- motivation as a standalone cause
- willpower as a standalone cause
- maturity
- character
- moral quality

These can be treated only as surface descriptions or social-language labels, not as structural causes.

---

## 10. Valid FCA Explanation Pattern

A valid FCA explanation must use this pattern:

```text
observable behavioral output
→ foreground-entry pattern
→ template source
→ solution-generation state
→ execution-chain state
→ feedback result
```

Chinese:

```text
可观察行为输出
→ 前台进入模式
→ 模板来源
→ 方案生成状态
→ 执行链状态
→ 反馈结果
```

---

## 11. Invalid FCA Explanation Pattern

Invalid:

```text
The person behaves this way because of personality or emotion.
```

Valid:

```text
The observed behavior results from a specific FCA chain: input foregrounding, template activation, solution-generation state, execution-chain condition, and feedback update.
```
